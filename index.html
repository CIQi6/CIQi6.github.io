什么是树？

树的组成:

  • 一组节点 (nodes)
  • 一组连接着节点的边 (edges)
    • 限制：任意两个节点只能有一条边

如上图，绿色标记的是树，红色的不是树。

树有什么用？

事实上，树结构是一个十分有用、高效的结构。在我们创建高效算法时非常有用：加快搜索速度、前缀、检查连接性等，例如 Search Trees，Tries，Heaps， Disjoint Sets。

另外，我们日常生活中的，大部分都是树结构。例如，一个族谱，文件目录。

树的遍历

  • 层次遍历 （Level order traversal）
  • 深度优先遍历 （Depth-First traversal）
    • 先序遍历 （Pre-Order traversal）
    • 中序遍历 （In-Order traversal）
    • 后序遍历 （Post-Order traversal）

层次遍历

层次遍历就是从上到下，从左往右一次遍历。

例如，上图的层次遍历序列为：D、B、F、A、C、E、G

先序遍历

先序遍历是先访问根节点，再访问左子树，访问右子树。

先序遍历序列为: D、B、A、C、F、E、G

中序遍历

中序遍历是先访问左子树，再访问根节点，最后访问右子树。

中序遍历序列为: A、B、C、D、E、F、G

后序遍历

后序遍历是先访问左子树，再访问右子树，最后访问根节点。

后序遍历序列为: A、C、B、E、G、F、D

什么是最小二分堆 (binary min-heap)

堆是一颗完整的树，而且满足最小堆性质。

  • 最小堆：每个节点都小于等于子节点。
  • 完整：仅在最底层有缺失值，且所有节点尽可能靠左。

最小二分堆的操作

  • add ： 将值暂时添加到末尾，“ Swim up” 到合适的位置。
    • swim up ：如果子节点小于父节点，则交换两个节点。
  • getSmallest ：返回根节点的引用
  • removeSmallest ：交换根节点和堆的最后一个节点，” Sink down “ 到合适的位置。
    • sink down ：如果父节点大于子节点，则交换两个节点。

copy constructor 的杂谈

copy constructor 为什么第一个参数必须为引用类型？

为了减少一次内存拷贝吗？

仔细想想，并不是这个原因。

The fact that the copy constructor is used to initialize nonreference parameters of
class type explains why the copy constructor’s own parameter must be a reference. If
that parameter were not a reference, then the call would never succeed—to call the
copy constructor, we’d need to use the copy constructor to copy the argument, but to
copy the argument, we’d need to call the copy constructor, and so on indefinitely.

这是 cpp Primer 对此的解释。

是的，避免 copy constructor 函数无限递归，最终造成栈溢出。

如果 copy constructor 第一个参数为非引用类型，即通过传值的方式。

为了调用 copy constructor ，我们需要把我们传给 copy constructor 的参数（例如： string(“999”) ），给 copy constructor 参数类表中， 又因为参数列表中的参数是未构造的，而我们又通过传值的方式传递。因此，我们又需要调用一次 copy constructor 。最终陷入无限递归。

在下面几种情况下会调用拷贝构造函数：

  • 显式或隐式地用同类型的一个对象来初始化另外一个对象。
  • 作为实参传递给一个参数为非引用类型的函数。
  • 在函数体内返回一个非引用类型对象时。
  • 初始化序列容器中的元素时。比如 vector<string> svec(5)。
  • 用列表的方式初始化数组元素时。string a[] = {string(“hello”), string(“world”)}; 。

tmux食用指南

什么是tmux

tmux是终端复用(teminal multiplexper)的简称。通常一个终端只能同时执行一个命令，例如，当你输入sleep 2000命令后，你必须要等终端执行完这条命令，你才能执行其他的命令。这显然十分的不便。一个解决方法是同时开多个终端。Bing！tmux就是这个原理。它可以让你实现多终端，并且在这些终端里灵活切换。

tmux的基本概念

tmux中有三个重要的概念：session、window、pane.

它们的结构如下：

——session00

​ ——window00

​ ——pane00

​ ——window01

​ ——window00

session包含window，window包含pane。

session类似我们的浏览器，window类似于浏览器的标签页，pane类似于网页里面的各个部件。

tmux命令总结/复习

  • session - 每个会话都是一个独立的工作区，其中包含一个或多个窗口
    • tmux 开始一个新的会话
    • tmux new -s NAME 以指定名称开始一个新的会话
    • tmux ls 列出当前所有会话
    • 在 tmux 中输入 <C-b> d ，将当前会话分离
    • tmux a 重新连接最后一个会话。您也可以通过 -t 来指定具体的会话
  • window - 相当于编辑器或是浏览器中的标签页，从视觉上将一个会话分割为多个部分
    • <C-b> c 创建一个新的窗口，使用 <C-d>关闭
    • <C-b> N 跳转到第 N 个窗口，注意每个窗口都是有编号的
    • <C-b> p 切换到前一个窗口
    • <C-b> n 切换到下一个窗口
    • <C-b> , 重命名当前窗口
    • <C-b> w 列出当前所有窗口
  • pane - 像 vim 中的分屏一样，面板使我们可以在一个屏幕里显示多个 shell
    • <C-b> " 水平分割
    • <C-b> % 垂直分割
    • <C-b> <方向> 切换到指定方向的面板，<方向> 指的是键盘上的方向键
    • <C-b> z 切换当前面板的缩放
    • <C-b> [ 开始往回卷动屏幕。您可以按下空格键来开始选择，回车键复制选中的部分
    • <C-b> <空格> 在不同的面板排布间切换
